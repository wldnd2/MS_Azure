Protocol이란 무엇인가요?
1. 통신 당사자만이 알아 볼 수 있는 약속
2. sender와 receiver, 중간 장치간의 소통을 위한 규칙
3. 컴퓨터 사이의 정보 전달과 관련된 약속
4. 네트워크 속도를 조정하는 규칙
정답: 2
해설: Protocol은 sender와 receiver, 중간 장치간의 소통을 위한 규칙입니다.

Protocol Layering을 이해하기 위한 두 가지 시나리오는 어떤 것이 있나요?
1. Maria와 Ann 사이의 Communication이 2개의 레이어를 사용하는 경우와 3개의 레이어를 사용하는 경우
2. Maria와 Ann 사이의 Communication이 오직 하나의 레이어에서만 발생하는 경우와 복잡한 Network에서 발생하는 경우
3. Communication이 무선인 경우와 유선인 경우
4. Communication이 Local Network인 경우와 Wide Area Network인 경우
정답: 2
해설: Protocol Layering을 이해하기 위한 두 가지 시나리오는 Maria와 Ann 사이의 Communication이 오직 하나의 레이어에서만 발생하는 경우와 복잡한 Network에서 발생하는 경우입니다.

protocol layering의 두 가지 원칙은 무엇인가요?
1. Listen/Speak, Send/Receive
2. Give/Take, Cipher text/Plain text letter
3. Listen/Speak, Enveloped mail/Plain text letter
4. Give/Take, Send/Receive
정답: 4
해설: Protocol layering의 두 가지 원칙은 각 레이어가 양방향 소통에 있어 두 가지의 opposite task를 수행할 수 있어야 하고, 양쪽 각 레이어의 두 객체는 동일해야(identical) 한다는 것입니다.

한 쌍인 레이어 간의 Logical connection은 무엇을 의미하나요?
1. 데이터 전송에 있어 물리적인 연결을 의미합니다.
2. 데이터 전송에 있어 서로 다른 레이어 간의 연결을 의미합니다.
3. 한 쌍의 레이어 간의 통신을 의미합니다.
4. 데이터 전송에 있어 전송 계층과 수신 계층 간의 통신을 의미합니다.

해설: 한 쌍인 레이어 간의 Logical connection은 레이어 간의 통신을 의미합니다.
